# Code of Conduct

## Our commitment

AI_SYSTEM_BUILDER is an open-source project built around clarity, discipline, evidence, and respectful technical collaboration.

We want this repository to be a welcoming space for contributors, reviewers, users, and maintainers regardless of background, experience level, identity, location, or viewpoint.

Participation in this project means helping keep the environment respectful, focused, and technically grounded.

---

## Expected behavior

Examples of behavior that supports a healthy project environment include:

- being respectful and constructive
- giving and receiving feedback in good faith
- explaining technical concerns clearly
- keeping discussions focused on the project
- asking questions when requirements or behavior are unclear
- respecting the project roadmap, architecture guardrails, and validation discipline
- assuming good intent while still requiring evidence for technical claims

---

## Unacceptable behavior

Examples of unacceptable behavior include:

- harassment, insults, intimidation, or personal attacks
- discriminatory language or behavior
- sexualized language, imagery, or attention
- trolling, repeated disruption, or bad-faith argument
- publishing private information without permission
- pressuring maintainers to bypass the roadmap, tests, or architecture rules
- repeatedly submitting broad, unrelated, or disruptive changes after being asked to stop

---

## Technical disagreement

Technical disagreement is welcome.

This project expects technical discussions to stay grounded in:

- code behavior
- tests
- documentation
- architecture guardrails
- roadmap alignment
- reproducible evidence

Disagreement should focus on the change, not the person.

---

## Enforcement

The project maintainer is responsible for clarifying and enforcing acceptable behavior.

Possible responses to unacceptable behavior include:

- asking for clarification or correction
- editing or closing issues or pull requests
- removing comments
- limiting participation in a discussion
- blocking a user from the repository when necessary

Enforcement decisions should be proportional to the behavior and focused on protecting the project environment.

---

## Reporting

If you see behavior that violates this Code of Conduct, you can report it by opening a private or direct communication channel with the maintainer when available.

If no private contact method is available, avoid escalating the conflict in public threads. Open a short issue only when appropriate and keep the report factual.

---

## Scope

This Code of Conduct applies to:

- issues
- pull requests
- discussions
- code review
- repository documentation
- any project-related interaction connected to AI_SYSTEM_BUILDER

---
